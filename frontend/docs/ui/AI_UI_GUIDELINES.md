# AI 智能平台 UI 规范（落地版）

## 品牌方向
- 视觉关键词：智能网络、实时计算、数据流、可解释性。
- 主色：`#1E90FF`；辅色：`#22D3EE`；强调：`#7C3AED`。
- 语义色：成功 `#34D399`、预警 `#F59E0B`、风险 `#F87171`。

## 信息架构
- `AI工作台`：工作台、驾驶舱。
- `数据与训练`：数据源、数据集、类别管理、训练、模型库。
- `智能应用`：识别、票据、洞察、报价、物流。
- `系统治理`：系统管理。

## 页面模板
- 首页（工作台）：平台状态 + 快捷操作 + 趋势图 + 任务时间线。
- 训练页：三段式流程（数据准备 -> 训练配置 -> 结果评估）。
- 识别页：左输入右结果，强调置信度与反馈闭环。
- 洞察页：页头统一为“智能数据洞察”，保持内容区可读优先。
- 驾驶舱：保持大屏风格，返回/操作控件固定在稳定区域。

## 组件体系（Design System）

### 基础组件
- `AiPageHeader`：统一页面标题、副标题、操作区。
- `RealtimeSignal`：实时状态信号灯。
- `InferenceMetric`：指标卡（值 + 趋势）。
- `ModelStatusCard`：模型状态卡（版本、准确率、状态）。
- `PipelineStepper`：流程步骤条。

### 组件使用规则
- 标题层级统一：一级页面使用 `AiPageHeader`。
- 状态一致性：实时状态必须通过 `RealtimeSignal` 呈现。
- 业务指标优先用 `InferenceMetric`，避免纯文本堆叠。

## 动效与交互规范
- 全局动效：粒子网络慢速流动 + 闪烁 + 连线（弱干扰）。
- 局部动效：卡片 hover 提升阴影与边框亮度（中强度）。
- 强反馈：训练完成/部署成功使用明显状态色和提示（强）。
- 鼠标交互：背景粒子支持吸引/排斥，但不抢表单操作焦点。

## 文案策略
- 避免纯技术描述，补充用户收益表达。
- 每个核心页面至少提供一个“下一步行动”按钮。
- 错误态文案包含可执行建议（例如“去模型库部署后重试”）。

## 验收指标（KPI）
- 品牌识别：首次访问 3 秒内识别为 AI 平台。
- 一致性：核心页面标题、卡片、状态组件风格统一。
- 任务效率：训练/推理关键路径点击数降低。
- 可读性：高密度图表与表格区仍保持清晰。
- 性能：开启全局动效后页面保持流畅。
