农产品供应链智能验收与管理平台
需求规格说明书（V1.0 最终版）
1. 项目概述
本系统面向中农发集团食材供应链、食堂配送、食材验收、采购定价、供应商管理场景，通过 AI 图像识别、OCR、爬虫、智能分析实现全流程自动化、防作弊、降成本、提效率。
2. 业务目标
实现食材验收无人化、少人化
杜绝作弊、以次充好、重量虚报
自动对账、自动结算、减少人工错误
实时监控菜价，智能推荐采购方案
自动生成食品安全台账，满足审计要求
3. 整体技术架构（锁死）
后端：PHP + Python 双引擎
前端：Vue PC 管理端 + Vue H5 移动端
数据库：MySQL
部署：Linux 服务器，无 GPU 要求
模型：全部轻量级，CPU 可实时推理
4. 功能模块清单（全部锁死）
模块 1：AI 表单识别（送货单 / 验收单）
技术方案：
使用 百度智能云表单识别 API
弃用 PaddleOCR（精度不足）
Python 负责调用、解析、结构化输出
自动提取字段：菜品名称、规格、单位、数量、重量、单价、金额、供应商、批次号、日期
业务功能：
上传图片 → 自动识别 → 自动回填表单
与系统订单自动比对
差异高亮提示
模块 2：AI 图像识别（摄像头 + 电子秤场景）
技术路线 100% 锁定，不可变更：
2.1 菜品分类识别
模型：MobileNetV2
任务：识别白菜、土豆、西红柿、蔬菜、肉类等
输出：菜品名称、置信度
2.2 新鲜度 + 缺陷检测（核心）
模型：基于 MobileNetV2 的迁移学习 + 多标签分类
输出：
新鲜度分数（0～100）
缺陷类型：正常、虫眼、腐烂、发霉、萎蔫、压伤、泡水、泥土过多
一张图输入，一次推理同时输出：
菜品名称 + 新鲜度 + 缺陷列表
2.3 识别流程
电子秤摄像头拍照 → 菜品分类 → 新鲜度 / 缺陷检测 → 自动填入验收记录
模块 3：智能验收与防作弊系统
重量异常校验（菜品与重量合理性判断）
品质自动打分，低于阈值自动拦截
重复照片检测（防作弊）
供应商历史次品率自动统计
自动生成验收异常报表
模块 4：菜价爬虫与智能分析
定时爬虫：北京新发地批发市场菜价
历史价格曲线、涨跌幅度计算
价格异常自动预警
供应商报价 vs 市场价对比
智能推荐替代品与采购策略
模块 5：自动对账与结算系统
三单自动比对：
送货单 ↔ 验收单 ↔ 系统订单
差异自动高亮
自动计算金额、损耗、应付款
一键导出 Excel 结算单
按供应商 / 按月 / 按项目汇总
模块 6：库存与损耗预测
基于近 7/15 天用量预测采购量
临期 / 过期提醒
菜品损耗率统计
供应商损耗排名
模块 7：工人极简 H5 操作端
三步操作：拍照 → 确认重量 → 提交
全自动识别，零录入
大按钮、简洁、适合工地 / 仓库现场使用
支持离线提交
模块 8：领导驾驶舱大屏
今日采购金额、总量、供应商排名
异常单、次品、价格异常实时展示
30 天价格趋势、损耗趋势
系统节省成本统计
供应商综合评分
模块 9：食品安全与台账自动生成
视觉缺陷识别（霉烂、虫眼）
产地、天气、农残公告爬取
每日食品安全台账自动生成
满足检查、审计、溯源要求
模块 10：供应商管理与智能派单
供应商档案：价格、品质、准时率、损耗率
自动评分、黑名单机制
每日推荐最优供应商
月度考核报表自动生成
