## 一、一句话定位

本平台是面向 **食迅产业链 / 数据运营 / 物流调度** 场景的 **「AI 训练 + 业务数据洞察 + 可视化指挥」** 一体化 Web 应用：同一套 FastAPI 后端承载 **模型训练与推理**、**只读业务库聚合 API**、**通义千问 Function Calling 对话助手**、**高德 / 北斗 / 摄像头等物联网接入**，前端以 **Vue 3 + Element Plus + ECharts + Cesium/Three** 呈现驾驶舱、天枢大屏、智能物流等模块。

## 二、总体架构

```mermaid
flowchart TB
  subgraph client [前端 SPA]
    V[Vue 3 + Vite + Vue Router]
    EP[Element Plus]
    EC[ECharts / Three / Cesium]
    ASST[悬浮 AI 助手]
  end

  subgraph api [FastAPI 统一网关]
    MAIN[main.py 挂载路由]
    INS[/api/insights/business]
    CHAT[/api/chat]
    LOG[/api/logistics]
    ML[/api/training · models · recognition · datasets]
    MAP[/api/map]
  end

  subgraph data [数据层]
    BIZ[(业务 MySQL 只读聚合)]
    SXW[(物流业务库)]
    SQLITE[(平台元数据 SQLite)]
  end

  subgraph ai [智能服务]
    DS[DashScope OpenAI 兼容 API]
    FC[Planner + Answerer + Tools]
  end

  V --> MAIN
  ASST --> CHAT
  CHAT --> FC
  FC -->|httpx 回环| INS
  INS --> BIZ
  LOG --> SXW
  ML --> SQLITE
  CHAT --> DS
```

**设计要点**

- **前后端一体交付**：生产环境可由单容器同时提供 `/api/*` 与静态 `dist`，降低运维面。
- **业务口径单一来源**：AI 工具层不重复写 SQL，通过 HTTP 调用 `insights_business` 白名单 SQL，保证大屏、洞察 Tab、助手 **同一套数字**。
- **弱网 / VPN 友好**：助手侧支持 **接口样例 + 表结构缓存**（catalog），离线演示不中断。

## 五、AI 业务助手

面向管理层的 **自然语言问数 + 报告生成** 能力，与 Claude 式对话 UX 对齐：

- **两段式**：Planner（意图 JSON）→ Answerer（多轮工具调用 + 结构化输出）。
- **十余个业务工具**：KPI、排行、趋势、热力、告警、行情、下钻、元数据自省、`generate_report`（日报/周报/月报/企划案/经营简报）等。
- **前端契约**：`<data_card>` JSON 卡片（含 ECharts 配置）、`<report_content>` Markdown 报告卡 + **多格式导出**。
- **体验增强**：SSE 阶段反馈、启发式 ETA、`server_elapsed_ms` 与端到端耗时展示、业务日 **Asia/Shanghai** 对齐、行业「下午晚间下单高峰」语境写入系统提示。
- **回归脚本**：`scripts/run_chat_regression.py` + 用例 JSON，可持续验收。

**会议话术示例**：「我们不是套壳 ChatGPT，而是把 **企业白名单 SQL 口径** 封成工具，让千问只能在我们允许的接口上取数，再生成卡片和 Word。」

## 六、部署与运维

- **本地 / 机房**：`docker compose up` 拉起 `agent` + `mysql`（可选），端口 **8000** 对外。
- **前端产物**：构建于 `frontend/dist`，由 FastAPI 挂载静态资源（深链回退 `index.html` 逻辑见 `main.py`）。
- **业务库**：通过 `INSIGHTS_MYSQL_*` 等环境变量指向只读从库或专线（具体以运维配置为准）。
- **AI**：`AI_PROVIDER` / `AI_API_KEY` / `AI_MODEL_*`；无 Key 时可降级 **mock 演示路径**。

---

## 七、安全与合规要点

1. **业务库只读**：洞察与助手工具层以聚合 API + 白名单表为主，降低误写风险。  
2. **PII 脱敏**：会员姓名、电话等在工具返回层做掩码（如 `tools.py` 中 `_mask_member_row`）。  
3. **密钥不入库**：地图、摄像头、大模型密钥均走环境变量；**本文档不含任何密码**。  
4. **对话不落业务库**：会话历史默认内存 TTL（`cachetools`），避免聊天记录与订单库混存（若二期要审计日志可单独设计）。
