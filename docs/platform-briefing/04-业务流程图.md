# 业务流程图（端到端）

以下流程强调 **平台在供应链数字化中的旁路位置**：消费数据、AI 处理、展示与稽核，**不替代**核心交易系统写库。

## 1. 主链路：登录后使用平台能力

```mermaid
flowchart LR
  U[用户] --> L{已登录?}
  L -->|否| LG[登录页]
  L -->|是| M[主界面]
  M --> W[工作台]
  M --> AI[数据集/训练/模型/识别]
  M --> D[数据洞察]
  M --> C[驾驶舱/天枢]
  M --> LO[智能物流]
  M --> DOC[票据识别]
```

## 2. 只读业务洞察 + 实时 GMV

```mermaid
flowchart TB
  subgraph Biz["客户业务系统（既有）"]
    ERP[ERP/交易系统]
    DB[(业务 MySQL)]
    ERP --> DB
  end

  subgraph Platform["AI Agent 平台"]
    API[insights_business API]
    POLL[live_gmv 轮询 Hub]
    WS[WebSocket /ws/live-gmv]
    UI[驾驶舱 / 天枢大屏]
    API --> UI
    POLL --> WS
    WS --> UI
  end

  DB -->|只读 SELECT| API
  DB -->|只读增量聚合| POLL
```

## 3. AI 训练—部署—识别

```mermaid
flowchart LR
  DS[数据集准备] --> TR[创建训练任务]
  TR --> RUN[train.py 子进程]
  RUN --> MD[模型入库]
  MD --> DEP[部署模型]
  DEP --> INF[识别 API]
  INF --> R[Top-K 结果]
```

## 4. 票据识别与比对（概念）

```mermaid
flowchart TB
  UP[上传票据图] --> REC[识别引擎]
  REC --> STR[结构化字段]
  STR --> CMP{与订单/规则比对}
  CMP --> OUT[差异报告 / 展示]
```
