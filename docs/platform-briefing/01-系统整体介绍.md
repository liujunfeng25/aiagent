# AI Agent 供应链数字化平台 — 系统整体介绍

> 文档版本：V1.0（基于代码库梳理）  
> 说明：本平台指 `aiagent/` 一体化工程（FastAPI + Vue3 + Docker 等），面向 **AI 能力沉淀** 与 **业务数据只读洞察**；**不替代**客户核心 ERP/交易系统中的下单、主数据维护等写操作。

---

## 1. 建设目标

- **数字化创新**：在统一门户内提供工作台、数据驾驶舱、天枢大屏、数据洞察与智能物流等能力，支撑「看得见、算得清、可演示」的运营视图。
- **AI 赋能供应链**：集成图像分类模型训练与部署、识别中心、票据/单据智能识别与比对等能力，将 AI 嵌入业务流程周边（验收、稽核、展示），而非侵入核心账务写库。
- **自主研发平台**：前后端、可视化与部署方案自主可控，可按央企/集团要求进行私有化交付与审计说明。

## 2. 系统边界（重要）

| 类别 | 说明 |
|------|------|
| **业务主库（洞察用）** | 通过只读账号连接客户/集团既有 MySQL，执行 **SELECT** 类查询与聚合；分析中心接口明确限制为 **仅 SELECT**。 |
| **平台自有库** | 本地 SQLite（`ai_agent.db`）承载数据集、训练任务、模型元数据、物流绑定与费用等 **平台侧** 数据。 |
| **物流数据源（SXW）** | 按租户切库连接食迅等业务库，用于车辆、设备、轨迹等物流模块（与 PHP 后台切库逻辑对齐）。 |
| **不写回业务订单** | 不在本平台内完成「下单、新增商品、报价、收货」等核心业务写入；平台定位为 **旁路赋能与展示**。 |

## 3. 技术栈

- **后端**：Python 3.11、FastAPI、SQLAlchemy、PyMySQL、PyTorch（训练/推理）、异步任务与 WebSocket。
- **前端**：Vue 3、Vue Router、Element Plus、ECharts、高德地图能力（驾驶舱/物流相关）、设计 Token 统一视觉。
- **部署**：Docker Compose（Agent 服务 + 可选本地 MySQL 卷）；生产可挂载数据卷持久化模型与 SQLite。

## 4. 主要用户场景

1. **管理端用户**：登录后使用工作台、数据集与训练、模型库、识别中心、票据识别、数据洞察、报价抓取、系统管理、智能物流等菜单能力。
2. **指挥/参观场景**：数据驾驶舱（`/cockpit`）、天枢大屏（`/tianshu`，静态资源可内嵌）用于 KPI、地图、实时链路与动效展示。
3. **技术/数据用户**：在「数据洞察」与 REST API 中查看订单趋势、区域分布、热力、KPI 汇总等；分析 SQL 仅限只读。

## 5. 对外叙事关键词（汇报/投标可用）

- 自主研发的一体化 **AI + 数据智能** 平台  
- **业务库只读解耦**，降低对生产交易系统风险  
- **训练—部署—推理** 闭环 + **单据智能识别**  
- **实时增量洞察**（如今日 GMV WebSocket 通道，依赖只读轮询与水印推进，零 DDL）
