# 核心逻辑说明

## 1. 双库策略：平台库 vs 业务库

- **SQLite（`ai_agent.db`）**  
  - 存储：数据集路径、训练任务、模型记录、操作日志、物流相关表（车辆、设备绑定、费用等）。  
  - **读写**：平台正常运行所必需，**不涉及**客户 ERP 中的订单主数据维护。

- **业务 MySQL（环境变量 `INSIGHTS_MYSQL_*`）**  
  - **用途**：数据驾驶舱、洞察 API、今日 GMV 实时通道等的数据来源。  
  - **约束**：分析接口 `POST /api/analysis/query` **仅允许** 以 `SELECT` 开头的语句；从架构上落实 **只读洞察**。

- **SXW MySQL（`SXW_MYSQL_*` + `SXW_MYSQL_SUPP_CODE`）**  
  - **用途**：智能物流模块，按 `supp_{code}` 租户库与 PHP 后台行为对齐。  
  - **连接**：`get_sxw_sessionmaker` 多库连接池缓存。

## 2. 业务洞察与 Schema 适配

- **白名单与元数据**：`app/business_insights/schema.py` 定义订单表、时间列、金额列等；`data_catalog.md` 描述表用途与环境变量覆写。  
- **弹性表名**：订单明细可能为 `orders_items` / `order_goods` 等；代码通过 `INFORMATION_SCHEMA` 与约定列名 **自动拼接 JOIN**，降低对接成本。  
- **典型 API**（节选）：日订单、会员排行、背单、区域/品类分布、地图客户点、侧栏智能摘要、KPI 汇总、日内 GMV/分钟单量、订单日历热力、运营告警等。

## 3. 今日 GMV 实时通道（`live_gmv_poller`）

- **目标**：在 **不改业务表结构（零 DDL）** 前提下，基于当日时间窗对订单表做 **增量轮询**。  
- **机制概要**：  
  - 维护内存水印（如最近 `add_time` / `order_id`），周期性查询新增/变更聚合。  
  - 启动时从库中 **同步当日累计 GMV 与单量** 初始化 Hub。  
  - **WebSocket** `/api/insights/business/ws/live-gmv`：新连接先推 **snapshot**；循环中对客户端 **ping** 保活；有新数据时 **broadcast** `batch`/`snapshot` 类消息。  
- **配置**：如 `INSIGHTS_LIVE_GMV_POLL_SEC` 控制轮询间隔。  
- **与前端**：驾驶舱 composable（如 `useCockpitLiveGmv.js`）与天枢大屏 hooks 消费同一通道，用于「实时链路」心电图节拍与数字刷新。

## 4. AI 训练与推理闭环

- **数据集**：目录型图像分类结构（按类别子文件夹）；支持统计图片数、训练快照对比「相对上次训练新增量」。  
- **训练**：`train_runner` 子进程调用 `train/train.py`，写状态 JSON；支持取消、查询进度。  
- **模型库**：记录路径与部署标记；启动时可按数据集去重、预置蔬菜模型注册。  
- **识别**：`inference.py` 加载 `model.pt` 与类别映射，返回 Top-K；支持指定 `model_id` 或当前已部署模型。

## 5. 票据 / 单据智能处理（`documents`）

- **识别**：上传图片 → 调用识别引擎（同步或 **SSE 流式** 进度）→ 返回结构化字段与 HTML 片段。  
- **比对**：支持与订单等结构化数据进行 **对比校验**（具体规则见路由与业务配置）。  
- **价值**：定位于 **稽核与效率**，不写回业务订单；结果由业务系统或人工采纳。

## 6. 智能物流（概要）

- 车辆列表、设备绑定（北斗/摄像头/温湿度等）、轨迹与地图展示、费用录入与统计。  
- 地图侧涉及 **坐标系转换**（如 BD→GCJ）、高德 Web 服务与前端 JSAPI 配置分离。

## 7. 静态资源与天枢大屏

- 生产构建后由 FastAPI 挂载 `frontend/dist`，深链回退 `index.html`。  
- `public/tianshu/` 可放置 **独立打包** 的大屏静态站，通过 `/tianshu` 或反向代理集成，便于与主平台 **同源 API** 调用。

## 8. 安全与合规注意（实施层面）

- 业务库账号应 **最小权限（只读）**；平台密钥仅通过环境变量注入。  
- 日志与导出需遵守集团数据分级与审计要求（实现上依部署策略扩展）。
